1195章 让我为你战斗到最后一刻（可视区ajax加载测试）


            依赖jQuery

            第一种用法   特点：只能传一个对象进行操作；可选择直接加载或者进入可视区加载
                （1） $('.obj').visible(boolean,function(){ ... });
                        对不同类名的容器是否进入可视区进行操作，回调函数可自行填写
                        this指向 .obj
                        第一个参数 boolean 可选。 true：表示进入可视区加载数据；false：表示不管是否可视直接加载
                （2）$('.obj').visible();
                        不传参数，返回 是否在可视区的 boolean值

             第二种用法   特点：可传一个数组的对象进行操作；只能使用进入可视区加载
                （1） $.visible(['Name1','Name2','Name3',...],function(){ ... });
                         对不同类名的容器是否进入可视区进行操作，回调函数可自行填写
                         this指向 当前进入可视区的 className 的对象
                         第一个参数 Array 必选。 可填写多个。只有一个的时候也要写成 ['Name']，否则无效
                （2）$.visible(['className1','className2',...]);
                        不传回调函数，返回 是否在可视区的 boolean值。
                        按填写顺序返回一个数组，如 [false,true,...]。
                        若只传了一个['className1']，直接返回 true 或者 false

                * 版本新增  $.visible(['Name',...],string,function(){ ... });
                         1、取消了第二种用法只能使用 类名 的设定。改为自行填全css选择器内容（增加实用性）
                         2、新增了一个参数，除了进入可视区加载外，到底部也可以加载数据（丰富功能）
                                新增参数的说明
                                        top：
                                            顶部进入可视区触发回调函数（适用于已知固定的多个模块进入可视区内的加载）；
                                        bottom：
                                            不可填错，到底部到可视区触发回调函数（适用于单个模块不确定加载数量的到底部加载新数据）
                                        不传回调函数的时候，也是一样返回 是否满足触发回调函数条件的boolean值
